⚡ Supabase 配置指南

本指南帮助你完成 Supabase 后端配置，实现真正的跨设备绑定（一码一设备）。

预计耗时：10分钟 | 费用：完全免费（Supabase 免费版：500MB 数据库、50,000 月活用户）

为什么从 CloudBase 切换到 Supabase？
CloudBase 免费套餐限制太多：不能添加安全域名（localhost 加不了）、不能使用静态网站托管。
Supabase 免费版没有这些限制，开箱即用，而且对前端直接调用非常友好。

📋 配置步骤

1注册 Supabase 账号

打开 supabase.com，点击右上角「Start your project」。

推荐使用 GitHub 账号登录（最方便），也可以用邮箱注册。

2创建项目

登录后进入 Dashboard，点击「New project」：

  • Organization：默认即可
  • Name：随意填写，如 mediation
  • Database Password：设置一个密码并记住它（后面建表要用）
  • Region：选择 Northeast Asia (Tokyo) 或 Southeast Asia (Singapore)（离中国近）

点击「Create project」，等待约 1-2 分钟 初始化完成。

3获取 API 密钥

项目创建完成后，进入项目 Dashboard → 左侧菜单 「Settings」 → 「API」。

复制以下两个值：

  • Project URL（格式：https://xxxxxxxxxxxx.supabase.co）
  • anon public key（以 eyJ... 开头的一长串字符）
💡 这两个值先保存到记事本，后面会用到。
4创建数据库表

进入左侧菜单 「SQL Editor」，点击「New query」，粘贴以下 SQL 并点击「Run」执行：

-- 创建设备绑定表 CREATE TABLE IF NOT EXISTS device_bindings ( id BIGSERIAL PRIMARY KEY, "codeHash" TEXT NOT NULL, devices JSONB DEFAULT '[]'::jsonb, "maxDevices" INTEGER DEFAULT 1, "activatedAt" TEXT, "updatedAt" TEXT ); -- 为 codeHash 创建索引（加快查询） CREATE INDEX IF NOT EXISTS idx_device_bindings_codehash ON device_bindings ("codeHash"); -- 开启 RLS（行级安全） ALTER TABLE device_bindings ENABLE ROW LEVEL SECURITY; -- 允许匿名用户读取所有记录（一码一设备的核心） CREATE POLICY "anon_can_read" ON device_bindings FOR SELECT TO anon USING (true); -- 允许匿名用户插入记录 CREATE POLICY "anon_can_insert" ON device_bindings FOR INSERT TO anon WITH CHECK (true); -- 允许匿名用户更新记录 CREATE POLICY "anon_can_update" ON device_bindings FOR UPDATE TO anon USING (true); -- 允许匿名用户删除记录 CREATE POLICY "anon_can_delete" ON device_bindings FOR DELETE TO anon USING (true);
💡 一次性粘贴全部 SQL 并执行即可，无需分步操作。
5开启匿名登录

左侧菜单 「Authentication」 → 「Settings」：

  • 找到 「Enable anonymous sign-ins」，打开开关
  • 点击页面底部的 「Save」
💡 这是关键步骤！不开启匿名登录，前端无法获取设备绑定数据。
6更新代码中的密钥

打开以下两个文件，搜索 SUPABASE_CONFIG，替换 URL 和 anonKey：

  • 人民调解协议生成器v6.0_方案二_设备绑定.html
  • 管理员面板v2_设备绑定.html
var SUPABASE_CONFIG = { url: 'https://xxxxxxxxxxxx.supabase.co', // ← 改成你的 Project URL anonKey: 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...', // ← 改成你的 anon key enabled: true };
7验证配置

用浏览器打开主页面（本地文件即可，Supabase 自动支持 localhost），按 F12 → Console 查看：

[Supabase] 初始化成功，项目: https://xxxxxxxxxxxx.supabase.co

✅ 看到这行日志，说明配置成功！

❌ 如果看到：

[Supabase] 初始化失败，回退到 localStorage 模式

说明配置有问题，请检查以上步骤。

🔧 常见问题

Q: 提示 "Anonymous sign-ins are disabled"？
A: 回到第 5 步，确认匿名登录开关已开启并点击了 Save。
Q: 数据库写入失败，报 403 或权限错误？
A: 检查第 4 步的 SQL 是否全部执行成功。特别是 RLS Policies，可以在左侧菜单「Authentication → Policies」中查看 device_bindings 表是否已有 4 条策略。
Q: 不想用 Supabase 了怎么办？
A: 在代码中将 enabled: true 改为 enabled: false，系统会自动回退到 localStorage 模式（仅限单浏览器）。
Q: 免费额度够用吗？
A: 完全够用。Supabase 免费版提供 500MB 数据库、5GB 带宽、50,000 月活用户。设备绑定数据量极小，几乎不会产生费用。

✅ 配置检查清单

  • 注册 Supabase 账号
  • 创建项目（选择亚洲地区）
  • 获取 Project URL 和 anon key
  • SQL Editor 中建表 + 设置 RLS 策略
  • 开启匿名登录
  • 更新两个 HTML 文件中的 SUPABASE_CONFIG
  • 打开页面验证控制台日志

🚀 测试跨设备绑定

  1. 在电脑A的Chrome中打开主页面，输入授权码 → 应显示「设备已绑定（第1/1台）」
  2. 在电脑B的Chrome中打开主页面，输入同一授权码 → 应显示「该授权码已被其他设备绑定使用」
  3. 在管理员面板中点击「解绑」，然后在电脑B重新输入 → 应成功绑定

配置完成后，设备绑定数据将存储在 Supabase 云端，所有设备共享同一份绑定状态。

☁️ CloudBase 配置指南

本指南帮助你完成 CloudBase 后端配置，实现真正的跨设备绑定。

预计耗时：5分钟 | 费用：完全免费（CloudBase 免费额度充足）

📋 配置步骤

1创建 CloudBase 环境

打开 腾讯云 CloudBase 控制台，点击「新建环境」：

  • 环境名称：随意填写，如 mediation
  • 计费方式：选择「按量计费」（有免费额度）
  • 地域：选择「上海」

创建完成后，复制环境ID（格式如 mediation-6gxq7o0k5c8e9e2f）

2开启匿名登录

进入 CloudBase 控制台，选择你刚创建的环境。

在左侧菜单找到：「登录授权」

⚠️ 如果左侧菜单没有「登录授权」，点击左下角的头像/设置图标切换到旧版 UI，或直接在环境卡片上点击进入环境。

在页面中找到「匿名登录」，将开关切换为「开启」

💡 新版 CloudBase 控制台可能在「环境设置 → 登录认证」中，如果找不到请切换到旧版控制台。
3配置安全域名（安全来源）

在 CloudBase 控制台，进入你的环境后，左侧菜单找到：

「环境配置」→「安全来源」

⚠️ 新版控制台路径：左侧 「环境」 菜单下，点击进入你的环境 → 上方标签页找 「安全配置」 或 「安全来源」

在「安全域名」区域，点击「添加域名」，添加以下域名：

  • localhost（本地测试）
  • 你的实际网站域名（如部署到线上，例如 www.qcloud.com 格式）
⚠️ 重要：127.0.0.1 这类纯 IP 地址不支持作为安全域名，会提示「域名格式错误」。
安全域名的正确格式示例：www.qcloud.com、www.qcloud.com:8080、*.qcloud.com
本地测试只需添加 localhost 即可（通常已默认添加）。
💡 注意：localhost 通常是默认自动添加的，可以确认一下是否已存在。
⚠️ 配置后约 1-2分钟 生效，最多可配置 50 个域名。
4创建数据库集合

在 CloudBase 控制台左侧菜单：「数据库」→「集合管理」→「新建集合」

集合名称填写：device_bindings

创建后，点击集合名称进入，切换到「权限设置」标签页，选择：

⚠️ 必须选：「所有用户可读写」

🚨 非常重要！如果权限选错（如「仅创建者可读写」），会导致：
· 浏览器A绑定的设备记录，浏览器B 读取不到 → 一码一设备机制完全失效
· 两个浏览器都能各自创建绑定，同一个授权码被多台设备同时使用

正确设置：选择「所有用户可读写」（匿名用户之间需要能读取彼此创建的记录）
5更新代码中的环境ID

打开以下两个文件，将环境ID替换为你的实际环境ID：

  • 人民调解协议生成器v6.0_方案二_设备绑定.html — 搜索 envId:
  • 管理员面板v2_设备绑定.html — 搜索 envId:
// 将这两处的 envId 替换为你的环境ID var CLOUDBASE_CONFIG = { envId: '你的环境ID', // ← 改这里 enabled: true };
6验证配置

用浏览器打开主页面，按 F12 打开开发者工具 → 控制台（Console），查看是否有以下日志：

[CloudBase] 初始化成功，环境: mediation-xxxxx

如果看到这行日志，说明配置成功！

如果看到：

[CloudBase] 初始化失败，回退到 localStorage 模式

说明配置有问题，请检查以上步骤。

🔧 常见问题

Q: 匿名登录失败？
A: 确保在「登录授权」中开启了匿名登录。注意匿名登录 token 有时效，刷新页面时会自动续期。
Q: 一台电脑刷新后自动登出？
A: 这是本地验证+云端验证的双重保护机制。v6.1已优化：本地验证通过后不再强制依赖云端验证结果，避免网络波动导致误登出。
如果仍有问题，F12打开控制台查看 [Auth] 开头的日志，确认是哪个环节触发了登出。
Q: 数据库操作报 403 错误？
A: 检查集合权限设置，确保设置了「所有用户可读写」。
Q: CORS 跨域错误？
A: 检查安全域名配置，确保添加了 localhost 或你的域名。配置后等10分钟生效。
Q: 不想用 CloudBase 了怎么办？
A: 在代码中将 enabled: true 改为 enabled: false，系统会自动回退到 localStorage 模式（仅限单浏览器）。

✅ 配置检查清单

  • 创建 CloudBase 环境
  • 开启匿名登录
  • 添加安全域名（localhost + 实际域名，不支持IP地址）
  • 创建 device_bindings 集合
  • 设置集合权限为「所有用户可读写」
  • 更新两个文件中的 envId
  • 打开页面验证控制台日志

🚀 测试跨设备绑定

  1. 在电脑A的Chrome中打开主页面，输入授权码 → 应显示「设备已绑定（第1/1台）」
  2. 在电脑B的Chrome中打开主页面，输入同一授权码 → 应显示「已绑定1台设备（上限1台），当前设备不在授权列表中」
  3. 在管理员面板中点击「解绑」，然后在电脑B重新输入 → 应成功绑定

配置完成后，设备绑定数据将存储在 CloudBase 云端，所有设备共享同一份绑定状态。
